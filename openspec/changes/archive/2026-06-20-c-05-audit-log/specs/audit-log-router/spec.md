## ADDED Requirements

### Requirement: GET /api/v1/auditoria returns a paginated, tenant-scoped audit log

The system SHALL provide `GET /api/v1/auditoria` that requires permission `auditoria:ver`. The endpoint SHALL enforce scope-based filtering: ADMIN and FINANZAS (`scope='all'`) see all entries for their tenant; COORDINADOR (`scope='own'`) sees only entries where `actor_id` matches their own `user_id`. All results SHALL be limited to the requesting user's tenant.

#### Scenario: ADMIN retrieves the full tenant audit log
- **WHEN** an authenticated user with the ADMIN role calls `GET /api/v1/auditoria`
- **THEN** the response SHALL be HTTP 200
- **AND** the response body SHALL conform to `AuditLogListResponse { items: [...], total: int, page: int, page_size: int }`
- **AND** all entries in `items` SHALL belong to the ADMIN's tenant

#### Scenario: FINANZAS retrieves the full tenant audit log
- **WHEN** an authenticated user with the FINANZAS role calls `GET /api/v1/auditoria`
- **THEN** the response SHALL be HTTP 200 with all tenant audit entries

#### Scenario: COORDINADOR retrieves only their own audit entries
- **WHEN** an authenticated user with the COORDINADOR role calls `GET /api/v1/auditoria`
- **THEN** the response SHALL be HTTP 200
- **AND** every entry in `items` SHALL have `actor_id == coordinador.user_id`
- **AND** entries generated by other actors SHALL NOT appear

#### Scenario: ALUMNO or PROFESOR receives 403
- **WHEN** an authenticated user with the ALUMNO or PROFESOR role calls `GET /api/v1/auditoria`
- **THEN** the response SHALL be HTTP 403 Forbidden

#### Scenario: Unauthenticated request receives 401
- **WHEN** `GET /api/v1/auditoria` is called without an Authorization header
- **THEN** the response SHALL be HTTP 401 Unauthorized

#### Scenario: Filtering by accion
- **WHEN** an ADMIN calls `GET /api/v1/auditoria?accion=IMPERSONACION_INICIAR`
- **THEN** the response SHALL be HTTP 200
- **AND** every entry in `items` SHALL have `accion == "IMPERSONACION_INICIAR"`

#### Scenario: Pagination with page and page_size
- **WHEN** there are 15 audit entries and an ADMIN calls `GET /api/v1/auditoria?page=2&page_size=10`
- **THEN** the response SHALL contain 5 items
- **AND** `total == 15`, `page == 2`, `page_size == 10` SHALL be reflected in the response body

#### Scenario: page_size is capped at 200
- **WHEN** `GET /api/v1/auditoria?page_size=500` is called
- **THEN** the response SHALL be HTTP 422 Unprocessable Entity (FastAPI validation rejects values > 200)
