## ADDED Requirements

### Requirement: First Alembic migration creates the tenant table

The system SHALL provide the first Alembic database migration (revision 001) that creates the foundational database schema for the `Tenant` model. This migration SHALL be the first in a series of versioned schema changes.

#### Scenario: Migration 001 creates the tenant table

- **WHEN** `alembic upgrade head` is executed
- **THEN** the `pgcrypto` extension is enabled (if not already present)
- **AND** the `update_updated_at_column()` trigger function is created
- **AND** a table `tenant` is created with columns: `id` (UUID PK with `gen_random_uuid()` default), `nombre` (VARCHAR NOT NULL), `codigo` (VARCHAR NOT NULL UNIQUE), `estado` (VARCHAR NOT NULL DEFAULT 'activo'), `created_at` (TIMESTAMPTZ NOT NULL DEFAULT NOW()), `updated_at` (TIMESTAMPTZ NOT NULL DEFAULT NOW()), `deleted_at` (TIMESTAMPTZ)
- **AND** a trigger `trg_tenant_updated_at` is created on the `tenant` table for automatic `updated_at` updates

#### Scenario: Migration 001 is reversible

- **WHEN** `alembic downgrade -1` is executed
- **THEN** the `tenant` table is dropped
- **AND** the trigger `trg_tenant_updated_at` is dropped
- **AND** the `update_updated_at_column()` function is dropped
- **AND** the `pgcrypto` extension is NOT dropped (to avoid affecting other potential dependents)

### Requirement: Migration uses async Alembic environment

The migration SHALL use the async Alembic environment configured in C-01, using `run_async()` with the async SQLAlchemy engine.

#### Scenario: Migration runs with async engine

- **WHEN** the migration is executed
- **THEN** it uses the async engine configured in `alembic/env.py`
- **AND** it completes successfully without synchronous fallbacks

### Requirement: One migration per schema change

Following project convention, each schema change SHALL be a separate Alembic migration. Migration 001 is exclusively for the `Tenant` model; future models will have their own migrations.

#### Scenario: Migration naming convention

- **WHEN** a new migration is created
- **THEN** its revision identifier follows Alembic conventions (autogenerated hash)
- **AND** the migration file includes a descriptive message (`"001_create_tenant"`)
